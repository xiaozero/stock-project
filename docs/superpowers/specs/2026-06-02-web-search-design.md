# 联网搜索功能设计

## 目标
为 AI 对话模块增加联网搜索功能，由模型（qwen3.5:0.8b）自动判断是否需要联网，并在需要时调用搜索 API 获取最新信息。

## 工作流程

```
用户发送消息
    ↓
模型首次响应，判断是否需要联网
    ↓
┌─────────────────────────────────────┐
│  模型需要联网                       │
│    ↓                                │
│  提取搜索关键词                     │
│    ↓                                │
│  调用 DuckDuckGo 搜索API            │
│    ↓                                │
│  将搜索结果追加到消息历史            │
│    ↓                                │
│  重新调用模型生成最终回答            │
│    ↓                                │
│  流式输出最终回答                   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  模型不需要联网                     │
│    ↓                                │
│  直接流式输出回答                   │
└─────────────────────────────────────┘
```

## 技术方案

### 1. 模型联网触发机制

qwen 模型在llama 中支持 tool calling，当需要联网时会：
- 输出包含 `tool_calls` 或类似格式的响应
- 或者输出特定 JSON 格式标记联网需求

**方案**：通过 System Prompt 引导模型输出联网标记：
```
当你需要搜索最新信息时，请先输出 <SEARCH>关键词</SEARCH>，然后继续输出你要搜索的内容。
```

### 2. 搜索 API

使用 **DuckDuckGo HTML 搜索 API**（免费，无需 API Key）：
- URL: `https://duckduckgo.com/html/?q=关键词`
- 解析返回的搜索结果摘要

### 3. 组件变更

#### 后端

**OllamaClient.java** - 修改 `chatStream` 方法：
- 检测模型输出是否包含 `<SEARCH>...</SEARCH>` 标记
- 如果检测到，暂停流式输出，触发搜索流程

**WebSearchClient.java** (新增)：
- 封装 DuckDuckGo 搜索功能
- 返回搜索结果列表（标题 + 摘要 + URL）

**ChatServiceImpl.java** - 修改 `chatStream` 方法：
- 处理联网搜索的完整流程
- 将搜索结果格式化为上下文，返回给模型再次生成

#### 前端

**ChatView.vue**：
- 添加"联网搜索"开关（默认开启）
- 用于调试或用户手动控制是否启用联网

### 4. 数据结构

**搜索结果格式：**
```json
[
  {
    "title": "搜索结果标题",
    "snippet": "搜索结果摘要...",
    "url": "https://example.com"
  }
]
```

**消息历史追加格式：**
```json
{
  "role": "user",
  "content": "搜索结果：\n1. 标题 - 摘要...\n2. 标题 - 摘要..."
}
```

## 风险与限制

1. **搜索结果解析**：DuckDuckGo HTML 页面解析可能因页面结构变化而失效
2. **模型能力**：qwen3.5:0.8b 为小模型，联网判断准确性可能有限
3. **搜索延迟**：联网会增加响应时间
